Minting a revival achievement from a vanilla advancement
  1. Gate: put the advancement's id (click to copy) in the entry's requires list — a living player must have earned it before this revival can be assigned. Match the entry's tier to the tier chip on the row.
  2. Task: design what the reviver must do, using the trigger tags on the row as inspiration. Supported patterns in scripts/gen_achievements.py: inv() hold-items-at-once (with count minimums and #tags), kill(), breed(), tame(), summon(), biome(), bare() for condition-less triggers, or vanilla_criteria("tab/slug") to copy a vanilla criteria block verbatim.
  3. Entry: add an ach(id, title, desc, icon, tier, criteria, requires=[...], repeatable=...) line to ROSTER in the script, then rerun it — it writes the definitions JSON and the backing advancement (parented into the in-game tab automatically).
  4. Verify: build and run the headless validator; the loader also logs and disables broken entries at server start.

Constraints: the criteria engine can't count kills ("kill 10 blazes" ⇒ use several distinct criteria or item-count predicates instead); every criterion listed is required; repeatable entries get their backing progress revoked after completion, so avoid criteria a player can't naturally redo.
